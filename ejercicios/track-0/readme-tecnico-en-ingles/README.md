# Ejercicio T0.1a — Escribe un README técnico en inglés

> **Modalidad: a mano, sin IA ni traductor.** Este ejercicio entrena tu **inglés técnico
> escrito**, el músculo que produce los READMEs de todos tus capstones. La IA corrige
> al final lo que tú ya escribiste — nunca lo escribe por ti.

**Track:** Track 0 — Empleabilidad, marca e inglés · **Lección:** `T0.1` Inglés técnico como GATE
**Ruta:** crítica · **Timebox:** 35 min

## 🎯 Objetivo

Producir un `README.md` **en inglés**, claro y con las convenciones de la industria, para
un proyecto real tuyo — distinguiendo el inglés "de traductor" (literal, raro) del inglés
técnico idiomático.

## 📋 Contexto

El **Definition of Done** de todo capstone del curso exige un **README en inglés**. Este
ejercicio es el molde que repetirás en F0 (CLI), F3 (API), F6 (RAG) y F7 (automatización
agéntica). Hacerlo bien una vez te ahorra hacerlo mal cinco veces.

## 📏 Primero-Sin-IA

1. Escríbelo **solo**, a mano, en inglés (timebox arriba). Sin Google Translate, sin LLM.
2. Si una frase te sale en español en la cabeza, anótala al margen y **tradúcela tú**,
   frase por frase. Ese esfuerzo manual **es** el ejercicio.
3. **Solo al final**, usa IA para *revisar y explicar* los patrones que fallaste.
4. Mañana, reescribe las secciones del README **de memoria**.

## 🛠️ Instrucciones

1. Elige **un proyecto real tuyo**, por mínimo que sea: tu repo de estudio, un script de
   10 líneas, o el plan de tu futura CLI de la Fase 0.
2. Copia `PLANTILLA-README.md` a un archivo nuevo llamado **`MI-README.md`** y rellénalo
   **en inglés**. No cambies los encabezados de sección.
3. Completa **`glosario.md`** (ya tienes `glosario.starter.md` como base): una tabla de
   **8 términos técnicos** que usaste o necesitaste, con la forma en español y la forma en
   inglés idiomático.
4. Lee tu `MI-README.md` **en voz alta**. Si te trabas leyéndolo, reescribe esa frase.

## ✅ Criterios de "hecho" (Definition of Done del ejercicio)

- [ ] `MI-README.md` tiene las 4 secciones (título + descripción de una frase,
      **Requirements**, **Usage** con el comando exacto, **Example** con salida realista).
- [ ] No hay traducción literal evidente (nada de "show in the screen", "for install",
      "execute the program" coloquial).
- [ ] Verbos técnicos correctos (run / build / install / returns / raises) y 3ª persona
      del singular bien conjugada ("it counts", no "it count").
- [ ] `glosario.md` tiene **8 entradas reales** (las que *a ti* te costaron, no genéricas).
- [ ] Puedes **leer el README en voz alta sin trabarte**.
- [ ] (Hilo transversal) Lo guardaste con un commit `docs: add english readme`.

## 💡 Pista (ábrela solo si superaste el timebox)

<details>
<summary>Mostrar pista</summary>

No empieces por la prosa: empieza por la **estructura**. Escribe primero los 4 encabezados
y luego la frase mínima de cada uno. Para la descripción usa la fórmula
*"A [tipo] that [verbo en 3ª persona] [objeto]"* — ej. *"A CLI tool that counts words in a
file."* Revisa la sección "del inglés de traductor al inglés técnico" de la lección antes
de pedir corrección.
